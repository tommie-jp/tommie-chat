# 検討事項: PWA

## 現状 (2026-03-31)

- manifest.json、sw.js を追加しPWA対応済み
- Service Worker: ネットワーク優先 + キャッシュフォールバック
- LAN環境（192.168.x.x等）ではSWを自動無効化（開発時のキャッシュ問題回避）
- sw.js の CACHE_VERSION を変更すると古いキャッシュが自動削除される

## ホーム画面への追加

- iOS/Android共に「ホーム画面に追加」はユーザーの手動操作が必須（セキュリティ制約）
- QRコードでURLは開けるが、ホーム画面への追加操作は省略できない
- Androidでは `beforeinstallprompt` でインストールダイアログを表示可能（iOSは非対応）

## 検討事項

### インストール促進バナー

- 初回アクセス時に「ホーム画面に追加するとフルスクリーンで遊べます」バナーを表示
- iOS: 共有ボタン → ホーム画面に追加 の手順を図解
- Android: インストールダイアログを自動表示
- 一度閉じたら再表示しない（Cookie/localStorageで管理）

### アプリアイコン

- 現在は favicon.png (32x32) をピクセルアート拡大した仮アイコン
- 正式なアプリアイコンのデザインが必要（192x192、512x512）

### クッキーマイグレーション

- 本番リリース後にクッキー構造を変更する場合、バージョンチェックで自動マイグレーションが必要
- 開発中はクッキー初期化（メニュー → 全パネル非表示）で対応可能

### オフライン対応

- 現在はサーバー接続必須のため、オフライン時は接続待ち画面のみ
- 将来的にオフラインモード（チャット履歴閲覧等）を検討可能
