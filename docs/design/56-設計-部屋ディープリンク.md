# 設計: 部屋ディープリンク（`?rm=` 直接参入）

URL ひとつで特定の部屋（ワールド）へ直接着地させる仕組みの設計メモ。

## 背景・目的

配信者やコミュニティ主が、X（旧Twitter）・Discord・配信概要欄などに
**部屋の URL を 1 本貼るだけで、ファンが DL 不要・顔出し不要で
その部屋に数秒で着地できる**ようにする。これが MMO 戦略上の
「人を連れてくる（B2C2C）」動線の生命線になる（詳細は戦略メモ参照）。

現状は `?ot=<ゲーム番号>` でリバーシ盤を開くディープリンクのみ実装済み。
これと同じパターンを流用し、`?rm=<部屋ID>` で部屋へ参入する版を追加する。

## URL スキーム（既存規約に準拠）

既存の URL スキーム規約（`docs/20-仕様書.md` の「URLスキーム規約」）で
既に予約されている記法に合わせる。**新規に `?room=` を作らず `?rm=` を使う**。

| URL | 動作 |
| --- | --- |
| `?rm=123` | 部屋（worldId）123 へ移動 |
| `?rm=123&pl=512,513` | 部屋 123 の座標 (512,513) へ移動 |
| `?rm=0` または指定なし | デフォルトワールド（worldId=0）へ |

将来の拡張（`docs/20-仕様書.md` 記載済み・本メモの対象外）:

- v3: パス形式 `https://mmo.tommie.jp/rm/123`
- v3: 部屋名スラッグ `https://mmo.tommie.jp/rm123`

> **メモ**: 現状 `worldId` は数値型（`createRoom` が採番した `worldId` を返す。
> [src/NakamaService.ts:844](../../src/NakamaService.ts#L844)）。
> 当面の `?rm=` は数値 ID をそのまま受ける。スラッグ対応は採番→名前の
> 解決 RPC が必要なので v3 へ後回し。

## 再利用する既存実装パターン

`?ot=` が踏んでいる **「URL 読取 → フラグ退避 → 接続後に実行」** の
3 ステップ遅延処理（[src/main.ts:27-42](../../src/main.ts#L27-L42)）を踏襲する。
ページロード完了から socket 接続確立までの遅延を吸収するための定石。

処理順序（`docs/20-仕様書.md` の「URLパラメータ読み取りのタイミング」と同型）:

1. ページロード時に `?rm=` を読み、`window.__pendingRoomId` 等の変数へ退避する
2. ページロード
3. `authenticateDevice`（localStorage or cookie → deviceId）
4. socket 接続確立 + 初期ワールド参加
5. 退避した `__pendingRoomId` があれば `changeWorldMatch(rmId)` を呼ぶ
   ([src/NakamaService.ts:534](../../src/NakamaService.ts#L534))
   - `?pl=x,z` があれば移動後に該当座標へ
   - 退避変数はメモリ保持のみ（localStorage 退避不要）

## 実装方針

### クライアント

1. [src/main.ts](../../src/main.ts) の `?ot=` 取り込みブロックの直後に
   `?rm=` / `?pl=` を読み取り、`window.__pendingRoomId` /
   `window.__pendingPos` へ退避する処理を追加する。
2. socket 接続・初期ワールド参加が完了する箇所（`joinWorldMatch` 後）で
   退避変数を確認し、`changeWorldMatch(rmId)` を呼ぶ。
   既存の部屋移動 UI（UIPanel の部屋リスト）と同じ経路を通す。
3. 履歴反映: 参入成功後に `history.pushState` で URL を `?rm={worldId}` に
   更新する（`?ot=` と同じく「戻る」で元に戻れるようにする）。

### 共有 UX

- 部屋内に「この部屋の URL をコピー」「X でシェア」ボタンを追加する。
- コピーする URL は `https://<host>/?rm=<worldId>`（必要なら `&pl=`）。
- 主（あるじ）がワンタップで貼れる形にするのが狙い。

## エラー・エッジケース

| ケース | 挙動 |
| --- | --- |
| `?rm=` の値が数値でない | 無視してデフォルトワールド（0）へ。`console.warn` を出す |
| 指定の部屋が存在しない / 削除済み | トースト「その部屋は見つかりませんでした」を表示し worldId=0 へ |
| 未ログイン（デバイス認証のみ）で開いた | そのまま参入可（`?ot=` と同じく匿名即入場を許容）。表示名は後から設定 |
| 満員（将来人数上限を設けた場合） | トースト表示し参入を拒否。デフォルトワールドへ |

`catch` でエラーを握りつぶさず、必ず `console.warn` 等でメッセージを出す
（プロジェクト規約）。

## セキュリティ

- `?rm=` は同一サーバ内の `worldId` 指定のみ。**URL でホスト/ドメインを
  変更させない**（別サーバ移動は別仕様・ホワイトリスト確認ダイアログ。
  `docs/20-仕様書.md` 参照）。フィッシング誘導を防ぐ。
- `worldId` はクライアントを信用せず、`changeWorldMatch` →
  `getWorldMatch` のサーバ側で存在・権限を検証する（既存の検証経路に乗せる）。
