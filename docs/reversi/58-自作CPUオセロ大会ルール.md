## オセロ大会のルール（公式記述ベース）

### 対戦形式

- 2台の自作CPUでオセロプログラムを動かし、片方が先攻、もう片方が後攻。
- CPU が出力した手を **人間が確認し、もう片方の CPU に入力する** ことで対戦を進める。
  - 通信プロトコルは規定なし。**人間が橋渡しする半自動方式**。
  - 入出力フォーマット（座標表記など）は各自任意で良いと読める。

### 持ち時間（最重要ルール）

1手あたりの思考時間上限は CPU のクロック周波数 `clock` [Hz] に応じて決まる。

```text
T_max = max(60 - 8 * log10(clock), 0.1)  [秒]
```

| クロック | 1手あたり最大時間 |
| -------- | ----------------- |
| 0.1 Hz   | 68秒              |
| 1 Hz     | 60秒              |
| 1 kHz    | 36秒              |
| 1 MHz    | 12秒              |
| 100 MHz  | 0.1秒             |

- 思考時間は **運営は計測しない／各プレイヤーが自己管理**。
- 表示処理の時間はカウント対象外（数秒以下推奨）。

### 大会の性質

> 「あくまでも本オセロ大会は厳密な試合ではなく、各々の自作CPUを使ったエンタメです」

厳密な競技規定ではなく、お祭り的イベント。

## 明記されていない項目（暗黙 or 任意）

connpass 上には以下の規定が **書かれていない**。

- 盤面サイズ（おそらく標準の 8×8 前提）
- 入出力フォーマット（座標 a1〜h8 か、行列インデックスか等）
- パス・終局・勝敗判定の細部
- 反則時の扱い
- 先攻後攻の決め方
- ハードウェアの定義（FPGA限定ではなく「自作CPU」全般。NAND CPU・三進CPU・Minecraftレッドストーン・フラッシュメモリ製など多様）

→ **「FPGA限定のオセロ大会」ではなく、自作CPU全般の交流イベント内のオセロ企画** という位置づけ。

## tommieChat 提案ポイント（メモ）

tommieChat のリバーシは Web/Babylon.js 実装なので大会本体には乗らないが、

- 標準リバーシルールの参照実装
- 対局プロトコルの叩き台（座標フォーマット・終局判定・パス処理）

として提案する価値がありそう。特に入出力フォーマットが未定義な点は、共通化提案の余地がある。
