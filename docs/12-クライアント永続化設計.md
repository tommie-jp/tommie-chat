# クライアント永続化設計

- 作成日: 2026-04-05

## 概要

クライアントの設定・状態は Cookie と localStorage / sessionStorage に保存される。
同一デバイス・同一ブラウザで複数タブからログインする場合の共有範囲に注意が必要。

## 保存先の使い分け

| 保存先 | 共有範囲 | 用途 |
| ------ | -------- | ---- |
| Cookie | 同一ドメインの全タブ | UI設定、テーマ、ログイン名 |
| localStorage | 同一ドメインの全タブ | デバイスID、prevSessionId（タブ固有キー付き） |
| sessionStorage | タブ固有（リロードで維持、新タブで新規） | tabId |

## Cookie 一覧

### チャット設定（チャット設定パネル）

| Cookie キー | 内容 | 全タブ共有 | 備考 |
| ----------- | ---- | ---------- | ---- |
| `chatOlMax` | 表示行数 | はい | UI表示設定 |
| `chatOlBgColor` | 背景色 | はい | UI表示設定 |
| `chatOlBgAlpha` | 背景透明度 | はい | UI表示設定 |
| `chatOlFontColor` | 文字色 | はい | UI表示設定 |
| `chatOlFontSize` | 文字サイズ | はい | UI表示設定 |
| `chatOlNameColor` | 名前色（自分） | はい | ※注1 |
| `chatOlTimeColor` | 時刻色 | はい | UI表示設定 |
| `chatOlWrap` | ワードラップ | はい | UI表示設定 |
| `chatOlShadow` | 影ON/OFF | はい | UI表示設定 |
| `chatOlShadowColor` | 影色 | はい | UI表示設定 |

## localStorage 一覧

| キー | 内容 | 共有範囲 | 備考 |
| ---- | ---- | -------- | ---- |
| `nakama_device_id_{loginName}` | デバイスID | loginName 単位 | Nakama 認証用 |
| `prevSessionId_{tabId}` | 前回セッションID | タブ固有 | ゴーストキック用 |

## sessionStorage 一覧

| キー | 内容 | 共有範囲 | 備考 |
| ---- | ---- | -------- | ---- |
| `tabId` | タブ固有ID | タブ固有 | prevSessionId のキー分離用 |

## 既知の制限事項

### ※注1: 名前色（chatOlNameColor）の共有

名前色は Cookie でデバイス全体に共有される。同一ユーザーであれば全セッションで
同じ色が使われるのが自然なため、現時点では許容する。

異なるユーザーIDで複数タブからログインした場合、名前色が共有される制限がある。
将来的に分離が必要な場合は Cookie キーに loginName を含めることで対応可能:

```text
chatOlNameColor → chatOlNameColor_{loginName}
```

## Google 認証後の DB 共有（2026-04-15 メモ）

Google OAuth でログインしたユーザーは、同一アカウントで複数端末からアクセスする
ケースが想定される。「ユーザーの分身」としての情報は DB（Nakama `storage` コレ
クション）に保存し、端末間で共有するのが自然。現在は表示名を DB 共有化中。

### DB 共有を強く推奨（アイデンティティ関連）

| 項目 | 現状 | 共有の理由 |
| ---- | ---- | ---------- |
| 表示名 | DB 共有化中 | 他人から見える自分 |
| アバター（sheetUrl + charCol + charRow） | 端末毎 | 他人から見える自分 |
| 名前色（`chatOlNameColor`） | Cookie（デバイス共有） | 他人のチャットに表示される自分の色。既知の制限あり（注1） |
| ブックマーク（部屋/位置） | 要確認 | 「マイお気に入り」。端末を乗り換えたら消えるのは不自然 |

### DB 共有を推奨（好み設定）

| 項目 | 現状 |
| ---- | ---- |
| テーマ（`uiTheme`） | Cookie |
| 言語設定（i18n） | Cookie |
| チャットオーバーレイ外観（背景色/透明度/文字色/サイズ/ワードラップ/影） | Cookie |

### デバイス固有のまま（DB 化しない）

- オーバーレイチャット位置・サイズ（`chatOlLeft` / `chatOlBottom` / `chatOlWidth` / `chatOlMax`） — 画面解像度依存
- ミニマップ位置・サイズ（`mmLeft` / `mmTop` / `mmSize` / `mmZoom`）
- デバッグツール位置・サイズ（`dbgLeft` / `dbgTop` / `dbgWidth` / `dbgHeight` / `dbgMin`）
- `devMode` / `showDebug`（開発用）
- `prevSessionId_{tabId}` / `tabId`（セッション内部）
- デバイスID（Nakama 認証用。Google 認証併用時も端末固有のまま）

### 設計上の注意点

1. **マージ戦略** — 初回 Google 認証時は DB が空。既存のローカル設定を DB に
   アップロードするか、DB 空なら無視するかを決める。例:
   - DB 空 → 端末の Cookie 値をアップロード
   - DB にある → DB 側を優先してローカルに反映
   - 両方ある → 最終更新時刻（mtime）で勝敗判定
2. **保存単位** — `storage` コレクションの構成案:
   - `user_profile`（1 レコード JSON）: 表示名・アバター・名前色・テーマ・言語・チャット外観設定をまとめる
   - `user_bookmarks`（1 レコード JSON 配列）: 可変長のため分離
3. **Cookie との二重管理** — Google 未認証時は Cookie、認証後は DB → Cookie に
   反映、Cookie 変更は DB に書き戻す同期ロジックが必要。
4. **優先順位** — インパクトが大きい順に着手推奨:
   1. アバター
   2. ブックマーク
   3. チャット外観設定・テーマ・言語
   4. 名前色
