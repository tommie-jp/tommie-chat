## Web向けMMO用Nakamaサーバー構築手順

Nakamaはスケーラビリティが高く、MMOのバックエンドとして非常に強力なオープンソースのゲームサーバーです。正確性を重視し、本番環境を見据えたDockerを用いた標準的な開発環境の構築から、クライアント接続までの手順を解説します。

### 1. 動作環境の準備

サーバーの起動とデータベース（PostgreSQLまたはCockroachDB）の管理を簡略化するため、Dockerを利用します。開発環境（ローカルPC）またはホスティング先のサーバーに以下のソフトウェアをインストールしてください。

* Docker
* Docker Compose

### 2. Nakamaサーバーのローカル起動

Heroic Labsが提供している公式のDocker Composeファイルを使用して、Nakama本体とデータベースを起動します。

1. 任意の作業ディレクトリを作成します。
2. 公式リポジトリから `docker-compose.yml` をダウンロードして配置します。
3. ターミナルで `docker-compose up -d` を実行します。
4. ブラウザで `http://127.0.0.1:7351` にアクセスし、初期認証情報（ユーザー名: admin / パスワード: password）で開発用コンソールにログインできることを確認します。

### 3. サーバーサイドロジック（モジュール）の開発

MMOではチート対策や状態の整合性を保つため、移動や戦闘の判定をサーバー側で行う「権威的マルチプレイヤー（Authoritative Multiplayer）」の実装が不可欠です。NakamaではGo、TypeScript、Luaを選択できますが、Web開発と親和性の高いTypeScriptを使用する前提のステップです。

1. サーバーコード用のディレクトリを作成し、Node.jsプロジェクトを初期化します。
2. 必要なパッケージ（`typescript`, `rollup`, `@heroiclabs/nakama-project` など）をインストールします。
3. エントリポイントとなるTypeScriptファイルを作成します。ここで、クライアントから呼び出せるRPC関数や、マッチのライフサイクル（参加、退出、メインループの毎フレーム処理）を定義するMatch Handlerを記述します。
4. コードをJavaScriptにビルドし、Nakamaコンテナが読み込むモジュールディレクトリ（通常は `data/modules`）に出力されるように設定し、サーバーを再起動（またはホットリロード）させます。

### 4. Webクライアントとの統合

HTML5/JavaScriptで作成したWebクライアントからNakamaサーバーに接続します。

1. Webプロジェクトに公式SDKパッケージ（`@heroiclabs/nakama-js`）をインストールします。
2. クライアントコード内でNakamaクライアントのインスタンスを生成し、ポート7350（デフォルトAPIポート）を指定します。
3. デバイスID認証、メールアドレス認証、またはカスタム認証を用いてプレイヤーをログインさせ、セッションを取得します。
4. リアルタイム通信用のWebSocket（Socketオブジェクト）を生成し、取得したセッションを使ってサーバーと接続します。これにより、MMOの要となる他プレイヤーとのステート同期やメッセージの送受信が可能になります。

### 5. 本番環境（デプロイ）への移行

ローカル開発が完了したら、Web上で公開するための本番環境を構築します。

* **データベースの分離**: 本番環境ではデータベース（CockroachDBまたはPostgreSQL）をNakamaノードとは別の専用サーバーに配置し、永続化とパフォーマンスを確保します。
* **SSL/TLS化とWSS**: Webブラウザ上（HTTPS）で動作するクライアントは、セキュリティ仕様上WSS（WebSocket Secure）接続が必須となります。NginxやHAProxyなどのリバースプロキシを前段に配置し、SSL証明書を設定してポート443経由でNakamaにルーティングします。
* **スケールアウト**: アクセス増加に合わせてNakamaのサーバーノードを複数立ち上げ、ロードバランサーでトラフィックを分散させます。

## サーバ起動するには

cd ~/nakama-project
docker compose up -d

## サーバ一時停止するには

コンテナを削除せず、処理だけを一時停止します。

cd ~/nakama-project
docker compose stop

## サーバ停止するには

ダッシュボードからはできないので、以下を実行
```bash
cd ~/nakama-project
docker compose down

```

## Nakamaの主要なポート番号

ポート 7350 (HTTP / WebSocket) : クライアント通信用（MMOのメイン）
