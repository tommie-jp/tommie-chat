# テストスクリプト一覧

テスト用シェルスクリプトの一覧とオプション。
全スクリプトは `test/` ディレクトリに配置。

前提: `cd nakama && docker compose up -d` でサーバ起動済みであること。

## リモートホストへのテスト

全テストスクリプトは `--host` / `--port` オプションまたは `NAKAMA_HOST` / `NAKAMA_PORT` 環境変数でリモートサーバへのテストに対応しています。
リモートホスト指定時は Go ビルド・サーバ再起動・Docker ログ収集が自動スキップされます。
テスト間のサーバ再起動もスキップされるため、前テストのプレイヤーが残った状態で次テストが実行されます。
各テストは固有のユーザー名を使うため大半のケースで問題ありませんが、厳密なテスト分離が必要な場合はローカルで実行してください。

| オプション / 環境変数 | 説明 | デフォルト |
| --------------------- | ------ | ----------- |
| `--host HOST` / `NAKAMA_HOST` | 接続先ホスト | `127.0.0.1` |
| `--port PORT` / `NAKAMA_PORT` | Nakama API ポート | `7350` |

```bash
# --host / --port オプション
./test/doAll.sh --host mmo.tommie.jp --port 443
./test/doTest-ping.sh --host mmo.tommie.jp

# 環境変数
NAKAMA_HOST=mmo.tommie.jp NAKAMA_PORT=443 ./test/doAll.sh

# 個別スクリプト
./test/doTest-snd-rcv.sh --host mmo.tommie.jp -n 100
./test/doTest-sustain.sh --host mmo.tommie.jp --port 443 -n 50 -t 60
```

`.test.ts` ファイルも `NAKAMA_HOST` / `NAKAMA_PORT` 環境変数に対応しているため、
vitest を直接実行する場合は環境変数で指定できます:

```bash
NAKAMA_HOST=mmo.tommie.jp NAKAMA_PORT=443 npx vitest run test/nakama-concurrent.test.ts
```

## 一括実行スクリプト

### `doAll.sh` — 全テスト一括実行

5つのテストスクリプトを順番に実行する。人数を段階的に増やして繰り返し実行も可能。

| オプション | 説明 | デフォルト |
| ----------- | ------ | ----------- |
| `-n N` | N人で1回実行 | 各スクリプトの既定値 |
| `--counts C` | カンマ区切り人数リストで順に実行（失敗で停止） | — |
| `-s S` | S人から段階実行（失敗で停止） | — |
| `--step D` | `-s` と併用。D人ずつ増加 | 100,500,1000,1500,2000,3000... |
| `--host HOST` | 接続先ホスト | `NAKAMA_HOST` or `127.0.0.1` |
| `--port PORT` | Nakama API ポート | `NAKAMA_PORT` or `7350` |
| `-v` | 全ログ出力（間引きなし） | off |

**実行テスト:**

| # | スクリプト | 内容 |
| --: | ----------- | ------ |
| 1 | doTest-concurrent-login.sh | 同時接続テスト |
| 2 | doTest-sustain.sh | 持続接続テスト |
| 3 | doTest-snd-rcv.sh | 送受信整合性テスト |
| 4 | doTest-player-list.sh | プレイヤーリスト通知テスト |
| 5 | doTest-ccu-db.sh | 同接履歴 DB永続化テスト |

**使用例:**

```bash
./test/doAll.sh                          # デフォルト人数で1回実行
./test/doAll.sh -n 2000                  # 2000人で1回実行
./test/doAll.sh --counts 100,500,1000,2000  # 段階実行（失敗で停止）
./test/doAll.sh -s 100 --step 100        # 100→200→300...と増加
./test/doAll.sh -s 100                   # 100→500→1000→1500→2000...
```

**ログ出力先:**

| ファイル | 内容 |
| --------- | ------ |
| `test/log/doAll-<テスト名>-YYYYMMDD-HHMMSS.log` | 各テストの全出力（ANSI除去済み） |
| `test/log/all-YYYYMMDD-HHMMSS.md` | Markdownレポート |
| `test/log/00-all.md` | 最新レポートへのシンボリックリンク |

### `doTest-reconnect.sh` — 再接続テスト

WebSocket切断→再接続後の状態復元を検証する。
2人のプレイヤーで切断→再接続を行い、AOI_ENTER・プロフィール・セッション管理が正しく動作するかを確認する。

| オプション | 説明 | デフォルト |
| ----------- | ------ | ----------- |
| `-h` | ヘルプ表示 | — |

**検証項目:**

| # | 内容 |
| --: | ------ |
| 1 | 切断→再接続で新sessionIdを取得 |
| 2 | 再接続後に他プレイヤーのAOI_ENTERを受信 |
| 3 | 再接続後に他プレイヤーが新セッションのAOI_ENTERを受信 |
| 4 | 再接続後にプロフィール取得が正常動作（自分・他プレイヤー双方向） |
| 5 | 古いsessionIdでプロフィール取得すると空で返る |
| 6 | 3回連続再接続しても正常動作 |

**手動テスト（ブラウザ DevTools）:**

コンソールから `game.nakama.socket.disconnect()` を実行すると即座にWebSocketが切断され、
RETRY→再接続の一連のフローをテストできる。

```text
確認手順:
  1. ブラウザでログイン（別端末で他プレイヤーも接続しておく）
  2. DevTools Console で game.nakama.socket.disconnect() を実行
  3. 右上バッジが RETRY（黄色）になることを確認
  4. 1秒後に ON（緑）に復帰することを確認
  5. 同接数・プレイヤーリストが正しいことを確認
  6. 他プレイヤーのアバターが再表示されることを確認
```

> DevTools の Network → Offline でもテスト可能だが、WebSocketハートビートタイムアウト（60秒）
> まで切断が検出されないため、`game.nakama.socket.disconnect()` の方が即座にテストできる。
