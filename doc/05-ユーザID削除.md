# ユーザID削除

Nakamaサーバで作成されたアカウントを削除する方法。

## コマンド

```bash
docker compose -f nakama/docker-compose.yml exec postgres \
  psql -U nakama -d nakama -c "DELETE FROM users WHERE custom_id != '';"
```

ポート5432が公開されているので、こちらでもOKです:

```bash
psql -h 127.0.0.1 -U nakama -d nakama -c "DELETE FROM users WHERE custom_id != '';"
# パスワード: localdev
```

## 1. 管理コンソール（最も簡単）

1. ブラウザで `http://127.0.0.1:7351` にアクセス
2. ログイン（デフォルト: `admin` / `password`）
3. **Players** → 該当ユーザーを選択 → **Delete**

## 2. REST API

### ユーザー一覧を取得

```bash
curl http://127.0.0.1:7351/v2/console/account \
  -u admin:password
```

### ユーザーIDを指定して削除

```bash
curl -X DELETE http://127.0.0.1:7351/v2/console/account/<user-id> \
  -u admin:password
```

## 3. PostgreSQL 直接操作

### コンテナに入る

```bash
docker exec -it <postgres_container> psql -U postgres nakama
```

### ユーザー確認

```sql
SELECT id, username, custom_id, create_time FROM users;
```

### custom認証で作成されたアカウントを全削除

```sql
DELETE FROM users WHERE custom_id != '';
```

### 特定ユーザーを削除

```sql
DELETE FROM users WHERE username = 'ユーザー名';
```

## 備考

- custom認証で作成されたアカウントを削除すれば、同じusernameでdevice認証の新規アカウントを作成できる
- `users`テーブルの`custom_id`列にはcustom認証のID、`device_id`列のリンクテーブル`user_device`にはdevice認証のIDが格納される
