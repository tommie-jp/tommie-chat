# DB: 同接データ（CCU履歴）

## 概要

サーバ終了時に1分間隔の同接数履歴をDBに保存し、再起動時に過去10日分を復元する。

## ストレージキー

| 項目 | 値 |
|---|---|
| Collection | `ccu_data` |
| Key | `history_1m` |
| UserID | `00000000-0000-0000-0000-000000000000`（システムユーザー） |
| PermissionRead | 0（非公開） |
| PermissionWrite | 0（非公開） |

## Value（JSON）

```json
{
  "timestamps": [1741500060, 1741500120, 1741500180, ...],
  "values":     [5, 3, 0, ...]
}
```

| フィールド | 型 | 説明 |
|---|---|---|
| `timestamps` | `int64[]` | 各サンプルのUnix秒（1分間隔） |
| `values` | `int[]` | その時点の同接数（1分間の1秒サンプル平均値） |

## データフロー

```
MatchLoop (毎秒)
  └─ appendCcu1sSample() → ccuHistory1s に追加（最大300件 = 5分）

MatchLoop (毎分)
  └─ flushCcu1mSample() → 1秒サンプルを平均化
       → ccuHistory1m + ccuHistory1mTs に追加（最大14400件 = 10日）

MatchTerminate (shutdown時)
  ├─ flushCcu1mSample() → 未フラッシュの1sデータを1mへ
  └─ saveCcuHistory1m() → Nakama Storage API で DB に書き込み

InitModule (起動時)
  └─ loadCcuHistory1m() → DB から読み込み、過去10日分のみメモリに復元
```

## メモリ上のデータ構造

| 変数 | 型 | 間隔 | 最大件数 | 期間 |
|---|---|---|---|---|
| `ccuHistory1s` | `[]int` | 1秒 | 300 | 5分 |
| `ccuHistory1m` | `[]int` | 1分 | 14400 | 10日 |
| `ccuHistory1mTs` | `[]int64` | 1分 | 14400 | 10日 |
| `ccu1sAccum` | `[]int` | — | — | 1分間の蓄積バッファ |

## Nakama設定（docker-compose.yml）

MatchTerminate が呼ばれるために必要な設定:

```
--shutdown_grace_sec 5    # Nakama: SIGTERM後のgrace period（秒）
stop_grace_period: 10s    # Docker: SIGKILLまでの猶予（Nakamaのgrace periodより長く）
```

## 内部テーブル

Nakama Storage API は PostgreSQL の `public.storage` テーブルに格納する。
主キーは `(collection, key, user_id)` の複合キー。CCU履歴は1レコードとして保存される。
