# UI改善メモ — デバッグ寄り→ユーザー寄り

- 作成日: 2026-04-04
- 対象バージョン: 0.1.14

現在の画面設計はデバッグ（開発者）寄りの要素が多い。
チャット主体のアバターSNSとして、一般ユーザー向けに改善できる項目をまとめる。

## すぐ改善できるもの

### 1. 右上の情報表示を簡素化

- 現在: `ON @user_73wl2y 4ms 60FPS` — 全部常時表示
- 提案: 通常は `ON` バッジだけ表示。ping/FPSはバッジクリックやデバッグツール開放時のみ表示
- 理由: 一般ユーザーは ping/FPS を気にしない

### 2. 左下の座標・方向を非表示（デフォルト）

- 現在: `(512, 512) ↑N` が常時表示
- 提案: デバッグツールのトグルで表示/非表示。デフォルトは非表示
- 理由: チャットアプリで座標を意識するユーザーは少ない

### 3. デバッグツールのメニュー名

- 現在: `02b.Theme`、`05.FPS`、`28.Player Pos` — 番号付き英語ラベル
- 提案: 番号を隠して日本語にする（「テーマ」「描画性能」「プレイヤー位置」）
- 理由: 番号は開発者の管理用で、ユーザーには意味がない

### 4. ハンバーガーメニューの項目整理（優先度高）

- 現在: 全機能がフラットに並んでいる（Ping グラフ、同接グラフ、デバッグツール等）
- 提案: 「設定」「開発者ツール」でグループ分けし、デバッグ系は折りたたみの中へ
- 理由: 一般ユーザーが使うのは表示名変更・チャット履歴・ユーザーリスト程度

## 中期的な改善

### 5. ログイン画面

- 現在: 画面下部にユーザーID入力欄が常時表示
- 提案: 初回は中央にモーダル表示。ログイン後は入力欄を非表示に
- 理由: ログイン済みなのに入力欄が見えるのは混乱する

### 6. ミニマップ

- 座標数値の代わりに、小さなマップにプレイヤーの点を表示
- チャットアプリなら「どこに人がいるか」が一目でわかるのが重要

### 7. チャットオーバーレイ設定

- 現在: `02c.行数`、`02d.背景`、`02e.文字` 等がデバッグツール内
- 提案: メニューから「チャット設定」として独立させる

## 開発者モード

通常モードと開発者モードを切り替えて、デバッグ要素の表示/非表示を制御する。

### 切り替え方法（A + B + C 併用）

- **A. URLパラメータ `?dev=1`** — 開発者がURLで即有効化。ブックマーク可
- **B. デバッグツール内トグル** — `00.開発者モード ON/OFF` を最上部に追加。Cookie保存で維持
- **C. 隠しコマンド** — ハンバーガーメニューのロゴを3回タップで有効化（Android開発者オプションに類似）

### モード別の表示内容

| 項目 | 通常モード | 開発者モード |
| ------ | ----------- | ------------- |
| 右上表示 | `ON` バッジのみ | `ON @名前 4ms 60FPS` |
| 左下座標 | 非表示 | `(512, 512) ↑N` |
| メニュー | 表示名/チャット履歴/ユーザーリスト/設定 | 全項目（Ping/同接/デバッグツール含む） |
| デバッグツール | なし（またはテーマ等の基本項目のみ） | 全項目（FPS/CPU/GPU/メモリ等） |

### 実装方針

- `body.dev-mode` CSSクラスで一括制御
- URLパラメータ `?dev=1` またはCookie `devMode=1` で判定
- CSSの `display: none` / `display: inline` で切り替え

## 優先度

すぐ効果が大きいのは **4（メニュー整理）** と **1（右上簡素化）**。
デバッグ情報を隠すだけで印象がかなり変わる。

開発中は現状のままの方が便利なので、
上記「開発者モード」トグルで切り替えられる設計を推奨。
