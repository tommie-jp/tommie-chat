# UI改善メモ — デバッグ寄り→ユーザー寄り

- 作成日: 2026-04-04
- 対象バージョン: 0.1.14

現在の画面設計はデバッグ（開発者）寄りの要素が多い。
チャット主体のアバターSNSとして、一般ユーザー向けに改善できる項目をまとめる。

## 実装済み（0.1.14）

### 1. 右上の情報表示を簡素化 ✅

- 通常モード: `ON` バッジのみ表示
- 開発者モード: `ON @名前 4ms 60FPS` 全て表示
- 各要素にツールチップ追加（バッジ/ユーザー名/ping/FPS）
- ホバーエフェクト（青背景＋赤文字）統一

### 2. 左下の座標・方向を非表示（デフォルト） ✅

- 通常モード: 非表示
- 開発者モード: `(512, 512) ↑N` 表示
- 座標オフセット表示（内部座標+512 → 0〜1023）
- 座標・方向にツールチップ追加

### 4. ハンバーガーメニューの項目整理 ✅

- 通常モード: プレイヤーリスト/チャット履歴/チャット設定/tommieChatについて/デバッグツール/表示名設定/ログアウト
- 開発者モード: 全項目（サーバ設定/サーバ接続ログ/Pingグラフ/同接グラフ/クッキー初期化 追加）

### 7. チャットオーバーレイ設定 → チャット設定パネル ✅

- デバッグツールから独立した「チャット設定」パネルを新設
- メニューから「チャット設定」で開閉
- 項目番号を 1〜7 に振り直し（02c/02d等のデバッグ番号を廃止）
  1. 表示行数（表示領域の高さを行単位で指定）
  2. 背景（色＋透明度）
  3. 文字（色＋サイズ）
  4. 時刻色
  5. 自分の名前色（サーバー経由で他ユーザーに共有）
  6. 折返
  7. 影（On/Off＋色）

### 開発者モード ✅

通常モードと開発者モードを切り替えて、デバッグ要素の表示/非表示を制御する。

#### 切り替え方法（A + B + C 併用）

- **A. URLパラメータ `?dev=1`** — 開発者がURLで即有効化。ブックマーク可
- **B. デバッグツール `00.DevMode`** — 最上部にOn/Offボタン。Cookie保存で維持
- **C. 隠しコマンド** — ハンバーガーメニューを3回タップで有効化

#### モード別の表示内容

| 項目 | 通常モード | 開発者モード |
| ------ | ----------- | ------------- |
| 右上表示 | `ON` バッジのみ | `ON @名前 4ms 60FPS` |
| 左下座標 | 非表示 | `(512, 512) ↑N` |
| メニュー | 基本項目のみ | 全項目（Ping/同接/サーバ設定等含む） |
| デバッグツール | 表示可（ブロックモードのため） | 表示可 |

### その他の実装済み改善

- 画面全体を `user-select: none` でコピー選択不可
- `backdrop-filter: blur` 削除（FPS改善）
- フッター（クレジット表記）にツールチップ・ホバー追加
- チャットオーバーレイの text-shadow を光彩に変更
- システムメッセージ（ログイン/ログアウト通知）のユーザー名に名前色適用
- 名前色のサーバー共有（OP_DISPLAY_NAME/OP_INIT_POS/OP_PROFILE_RESPONSE）
- 名前色変更時のデバウンス（300ms）
- 名前色の初期値をランダムパレットから選択
- 表示行数の仕様変更（テキスト行数ベースで判定、折り返し考慮）

## 未着手

### 6. ミニマップ

- 座標数値の代わりに、小さなマップにプレイヤーの点を表示
- チャットアプリなら「どこに人がいるか」が一目でわかるのが重要

#### 参考: 一般的なMMOミニマップの仕様

##### 配置

- **右上**が業界標準（WoW、FF14、黒い砂漠、Guild Wars 2、ESO）
- チャット欄（左下）やスキルバー（下中央）と干渉しない位置
- tommieChat ではチャットが左下〜下部にあるため、右上が自然

##### ズームUI

- `+` `-` ボタンがミニマップの枠内または直下に配置されるのが最も一般的
- PC: マウスホイールでもズーム可（ボタンと併用）
- モバイル: ピンチイン/アウト or ボタンのみ
- ズーム段階は3〜5段階が一般的
- tommieChat ではチャット主体のため2〜3段階で十分

##### 自分の位置

- プレイヤーは常にマップ中央に固定、移動すると周囲がスクロール
- 回転式（進行方向が常に上）と固定式（北が常に上）の2方式がある

| 方式 | 採用例 | 特徴 |
| ---- | ------ | ---- |
| 回転式 | WoW、黒い砂漠、GTA | 進行方向が常に上→直感的 |
| 固定式（北が上） | FF14、メイプル | 方角が安定→地理を覚えやすい |
| 切替可能 | ESO、Guild Wars 2 | 設定で選べる |

- tommieChat では固定式（北が上）がシンプルで十分

##### プレイヤーの表示

| 対象 | 形 | 色 | 備考 |
| ---- | --- | --- | ---- |
| 自分 | 矢印（▲） | 白 | 向きがわかる、他より少し大きい |
| 他プレイヤー | 小さい丸（●） | 緑/水色 | WoW、FF14 等で標準 |
| パーティメンバー | 丸 or 菱形 | 青 | WoW、ESO |
| フレンド | 丸 | 黄/オレンジ | 黒い砂漠 |
| NPC | 丸 or アイコン | 黄/緑 | FF14 |
| 敵対プレイヤー | 丸 | 赤 | WoW（PvP） |

- 自分は他より大きく、形も異なる（矢印 vs 丸）で差別化
- 色で関係性を区別するのが共通ルール

##### tommieChat 向け推奨

- 自分: 白い矢印（▲）— 向きがわかる
- 他プレイヤー: 緑の小さい丸（●）— シンプル
- PvPがないので赤/青の区別は不要
- 将来フレンド機能追加時に色分けの余地を残す

## 既知の不具合パターン: パネルのスクロールバーと WebGL キャンバス

### 症状

パネル内にスクロールバーが表示されると、スクロールバーの track（背景部分）が透明なため、
背後の WebGL キャンバス（空の色など）が縦ラインとして透けて見える。
パネルを移動するとラインも一緒に移動する。

### 原因

ブラウザのデフォルトスクロールバーは background が transparent。
パネルの背景が `rgba(255,255,255,0.88)` 等の半透明でも、スクロールバー部分は別レイヤーで
描画されるためパネル背景が適用されず、直接キャンバスが見える。

### 対処法

`overflow-y: auto` を持つ要素に `::-webkit-scrollbar` スタイルを追加する。

```css
#xxx-body::-webkit-scrollbar { width: 8px; }
#xxx-body::-webkit-scrollbar-track { background: rgba(255,255,255,0.95); }
#xxx-body::-webkit-scrollbar-thumb { background: rgba(0,0,0,0.25); border-radius: 4px; }
#xxx-body::-webkit-scrollbar-thumb:hover { background: rgba(0,0,0,0.4); }
```

### 該当パネル（対応済み）

- `#user-list-wrap` — ユーザーリストパネル
- `#bookmark-body` — ブックマークパネル

### 注意

**新しいパネルを追加する際は、`overflow-y: auto` を使う場合に必ずスクロールバーのスタイルを追加すること。**
この不具合は繰り返し発生しているため、チェックリストに含めること。
