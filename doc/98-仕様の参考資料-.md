# 仕様の参考資料

## 表示名の最大文字数

tommieChat の表示名上限を決める際の参考資料。

| ゲーム | 最大長 | 備考 |
| --- | --- | --- |
| Minecraft (Java) | 16文字 | 英数字・アンダースコアのみ |
| Final Fantasy XIV | 20文字 | 名+姓(各15文字) |
| World of Warcraft | 12文字 | |
| Fortnite | 16文字 | |
| Genshin Impact | 16文字 | |
| Roblox | 20文字 | |
| Guild Wars 2 | 19文字 | |
| PSO2 | 12文字 | |

### tommieChat の表示名設定

- 最大: **20文字**
- 最小: **0文字**（空欄の場合は `@ユーザID` を青色で表示）
- 制御文字は使用不可

日本語の場合、漢字1文字で2〜3文字分の情報量があるため、20文字あれば日本語でも英語でも十分な長さ。

### 表示名未設定時のフォールバック表示

| アプリ | 表示名未設定時 |
| --- | --- |
| Discord | ユーザー名をそのまま表示 |
| Twitter/X | `@username` を表示 |
| Slack | ユーザー名を表示 |
| LINE | 「未設定」と表示 |
| FF14 | 表示名必須(未設定不可) |
| Minecraft | ユーザー名 = 表示名(別設定なし) |

tommieChat では Twitter/X 方式を採用: `@ユーザID` をオレンジ色(`#ff9933`)で表示。

## モバイルアプリの複数行入力方式

セリフ入力のスマホ対応を決める際の参考資料。

| アプリ | Enter の動作 | 送信方法 | 備考 |
| --- | --- | --- | --- |
| LINE | 改行 | 送信ボタン | 設定で「Enterで送信」に変更可 |
| Discord | 送信 | Enter | Shift+Enter で改行(PCと同じ) |
| WhatsApp | 改行 | 送信ボタン | |
| FF14 (スマホ版) | 改行 | 送信ボタン | |
| Slack | 改行 | 送信ボタン | 設定で変更可 |
| iMessage | 改行 | 送信ボタン | |

主流は「Enter=改行、送信ボタンで送信」。Discord だけが例外的に Enter=送信。

### tommieChat のセリフ入力設定

- **PC**: Enter で送信、Shift+Enter で改行
- **スマホ**: Enter(改行キー)で改行、送信ボタンで送信

## 自動ログイン

ページを開くだけで即参加できる仕組み。

### 他サービスの初回ログイン方式

| アプリ | 初回ログイン | 方式 |
| --- | --- | --- |
| アメーバピグ | Amebaアカウント必須 | アカウント登録→ログイン |
| Gather.town | メール or Google認証 | アカウント必須 |
| spatial.chat | 名前入力だけで即参加 | ゲスト方式 |
| Figma | ブラウザで即プレビュー、編集はアカウント必須 | 段階的 |
| agar.io 等の io ゲーム | 名前入力→即プレイ | ゲスト方式 |

### tommieChat の自動ログイン設定

spatial.chat / io ゲーム型のゲスト即参加方式を採用。

| URL | 動作 |
| --- | --- |
| `localhost:3000` | 自動ログイン（ゲストモード） |
| `localhost:3000?login` | 手動ユーザID入力画面を表示 |

#### 自動ログインの流れ

1. ページロード
2. Cookie `loginName` を確認
   - **あり** → 保存済みユーザIDで自動ログイン
   - **なし** → ランダムID（`user_` + 6桁英数字、例: `user_a3x9k2`）を生成して自動ログイン
3. ログイン成功 → ユーザID行は非表示、ハンバーガーメニューに「ログイン画面」ボタン表示
4. ログイン失敗 → ユーザID入力行を表示してリカバリ

#### 手動ログイン画面へのアクセス方法

- URL に `?login` パラメータを付与
- ハンバーガーメニューの「ログイン画面」ボタン（ログイン後に表示）

#### 認証方式

- Nakama Device Auth（パスワード不要）
- デバイスID は `localStorage["nakama_device_id_{username}"]` に自動保存
- 同じブラウザ・同じユーザIDなら再ログイン時に同一アカウントとして認識
