# スマホ表示テスト

Chrome DevTools のデバイスエミュレーションで確認する際の参考端末サイズ。

## 推奨テスト端末

| 端末 | 幅 | 用途 |
| ---- | --- | ---- |
| iPhone SE | 375 x 667 | 最小幅テスト（現行販売中iPhoneで最小） |
| iPhone 14/15 | 390 x 844 | 標準的なiPhone |
| iPhone 14/15 Pro Max | 430 x 932 | 大画面iPhone |
| Galaxy S系 | 360 x 800 | Android最小級（SEより小さい） |

## 最低テスト基準

| ターゲット | 最低幅 | 理由 |
| ---------- | ------ | ---- |
| 日本向け | 375px（iPhone SE） | 日本はiPhone比率が高く、これで大半をカバー |
| グローバル | 360px（Galaxy S/A系） | Android約72%のグローバル市場では360pxが最小実用幅 |

- 320px（古い低価格Android）は新興国に残存するが、切り捨ても現実的
- 375px でレイアウトが崩れなければ、ほぼすべての端末で問題ない

### グローバル市場の参考データ（2025年時点）

- **Android 約72%** / **iPhone 約28%**
- 東南アジア・インド・アフリカでは安価なAndroid（360px幅）が主流
- 北米・欧州・日本ではiPhoneが多い

| 幅 | 端末例 | 地域 |
| --- | ------ | ---- |
| 320px | Galaxy J1, 古い低価格Android | 新興国にまだ残存 |
| 360px | Galaxy S/A系, Xiaomi Redmi系 | 世界的に非常に多い |
| 375px | iPhone SE/6/7/8 | 日本・北米・欧州 |
| 393px | Pixel 7, Galaxy S24 | 現行Android標準 |

## メディアクエリ

本プロジェクトのスマホ判定CSS:

```css
@media (pointer: coarse) and (min-resolution: 2dppx) { ... }
```

- `pointer: coarse` — タッチデバイス
- `min-resolution: 2dppx` — 高DPIスクリーン（Retina等）

DevTools でエミュレーション時はこの条件に自動的にマッチする。

## テキストサイズとブラウザ差異

iPhone Safari と Chrome でテキストサイズが異なる場合がある。

- **Safari**: iOS本体の設定（設定 → アクセシビリティ → 画面表示とテキストサイズ → さらに大きな文字）に従う
- **Chrome**: iOS本体の設定を無視し、Chrome独自の設定を使う

### Chrome でテキストを大きくする方法

Chrome → 設定 → ユーザー補助機能 → テキストの拡大と縮小 → スライダーで調整

### 既知の制限

テキストサイズを拡大すると入力欄やボタンの高さが合わなくなりデザインが崩れる。これはフォントサイズだけが拡大され、UIコントロールのサイズ（px固定）が追従しないため。完全対応には全UIを `rem` ベースに書き直す必要があり、現時点では未対応。デフォルト設定での表示を優先する。

## 開発時の反映確認

クレジット表記にページ読み込み時刻（`#HHMM` 形式）が表示される（開発モード時のみ）。コード変更後にリロードして数字が変わっていれば最新コードが反映されている。
