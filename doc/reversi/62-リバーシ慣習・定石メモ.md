## 1. プロトコル系（他のリファレンス実装）

| 名前 | 概要 | 本書との関係 |
| ---- | ---- | ------------ |
| **NBoard Protocol** | NBoard（WZebra 後継の GUI）と Othello エンジンの通信仕様。テキストベース、`go` / `move` / `new` / `ping` / `status` 等 | 本書とほぼ同じ設計思想。`PI`/`PO` ≈ `ping`/`pong`、`SB`/`SW` ≈ `new`、`MOd?` ≈ `move` 相当。1 行 1 コマンド・ASCII・LF 区切りも同じ |
| **GGS (Generic Game Server) protocol** | 昔の Othello オンライン対戦サーバーのプロトコル | Thor 棋譜 DB と接続する実装が多い |
| **GTP (Go Text Protocol)** | 囲碁版の同種プロトコル。Chess の UCI に相当 | 「CPU は純粋リアクティブ」設計思想はここから輸入されている |

NBoard Protocol は仕様が公開されており、CPU 製作者にとっては本書とあわせて
読むと実装の参考になる。

## 2. 棋譜表記の慣習

- **Thor 棋譜形式**: WOF 公式 DB。60 手を `f5d6c3d3c4f4...` のように
  **全小文字・区切りなし**で連結。60 手未満は残りを `--` でパディング
- **大文字小文字の流派**:
  - 全小文字（Thor / Edax / 現代標準）← 本書もこちら
  - 黒大文字・白小文字（古い棋譜）
- **パスの表記**: `--` や `ps` や `PA`（流派により異なる。本書は `PA` 流派に準拠）
- **主要エンジン**: Edax / Logistello / Zebra。棋譜入出力はどれも Thor 形式準拠

**tommieChat への示唆**: 将来 tommieChat 内で棋譜保存・共有機能を付けるなら、
Thor 形式をエクスポート対応すると既存の解析ツールにそのまま流せる。

## 3. 定石名（対局表示に使える）

黒初手 `f5` から始まる主要定石:

| 定石名 | 代表的な出だし |
| ------ | -------------- |
| **虎定石**（とら） | `f5d6c3d3c4f4...` 最も有名 |
| **兎定石**（うさぎ） | `f5d6c3d3c4b4...` |
| **牛定石**（うし） | `f5d6c4...` |
| **鼠定石**（ねずみ） | `f5f6...` |
| **猫定石**（ねこ） | `f5d6e3...` |
| **蛇定石**（へび） | `f5f4...` |
| **縦取り** | 黒が縦方向に並べる戦型 |
| **斜め取り** | 斜め方向に並べる戦型 |

**tommieChat への示唆**: リバーシ UI で「観戦中の対局が○○定石に入りました」
という表示をすると観戦の楽しみが増える。NPC アバターが「虎だ！」とつぶやく演出も可能。

## 4. 対局運用の常識

1. **黒先白後・色交代**: 2 局 1 セットで色を交代して戦うのが普通（先手有利なので）。
   **再戦機能は色自動反転がデフォルト**
2. **持ち時間**: WOF 公式は 30 分切れ負け（全 60 手想定）。大会によっては
   25 分＋ 1 手ごとフィッシャー加算など
3. **引き分け有り**: 32-32。チェスと違って「引き分けに持ち込む」戦略は稀
4. **合法手がない時は強制パス**: 両者連続パス＝終局。片方だけパスは続行
5. **終局計算**: 勝者が空マスを総取りする流派（Stone Difference）と、
   実際の石だけ数える流派（Disc Count）がある。**公式は Disc Count**
6. **コーナー優位**: 四隅は一度取ったら裏返らない。初心者向けに
   「角取り警告」を出す UI は鉄板
7. **偶数理論**: 終盤の手数が偶数か奇数かで有利不利が変わる（上級者向け）

**tommieChat への示唆**: 対局 UI に「合法手ハイライト」「角をハイライト」
「パス強制時の自動通知」を入れるのは標準機能。

## 5. tommieChat への具体的な応用アイディア

1. **対局ルーム＝既存の部屋システムの一種**: 部屋タイプに `reversi` を追加、
   2 人入室で対局開始（[doc/50-設計-部屋システム.md](../50-設計-部屋システム.md) 拡張）
2. **観戦者 UI**: Babylon.js で 3D 盤面、周りのアバターが観戦。同接 100 人規模の
   対局配信が売りにできる
3. **定石名自動検出**: サーバー側で Thor DB（または定石パターン辞書）と照合し、
   定石名を chat に自動投稿（NPC が「それは虎定石ですね」とつぶやく）
4. **外部 CPU 参加**: [61-UARTプロトコル仕様.md](61-UARTプロトコル仕様.md)
   のプロトコルで、自作 CPU を 1 プレイヤーとして登録。表示名は `VE` 応答の
   識別名をそのまま使う
5. **棋譜ダウンロード**: 対局終了時に Thor 形式で書き出し、「再生」ボタンで
   時系列再現。SNS 投稿用スクリーンショットも生成
6. **レーティング**: WOF 公式は Elo ベース。同接 1000 人規模なら独自レートが必要
7. **色選択 UI**: 「先手 / 後手 / おまかせ」の 3 択。おまかせ＝コイン表示アニメーション
