     │  onAvatarInitPos(B2)                                  │

     │                                                       │

     onMatchPresenceLeave(B1)

     A が B1 の MatchLeave 通知を受け取れないケースがある:

## 問題のタイミング詳細（ケース a: ゴーストが永久に残る）

```

─────────────────────────────────────────────────────────────
t=0                                                   リロード
t=0.1                                                 pagehide → leaveMatch 失敗
t=0.5                                                 B2: login + joinChat
t=0.5  presenceJoin(B2) 受信                          
t=0.5  sendInitPos(A→B2) ※1
t=0.8                                                 B2: joinWorldMatch
t=0.8                                                 B2: onmatchdata 登録
t=1.0                                                 B2: joinMatch
t=1.0  matchPresenceJoin(B2)

t=2.0                          B1 の切断検知（※2）
t=2.0                          MatchLeave(B1)
t=2.0                          AOI_LEAVE(B1) → A へ送信

       ここで A が受信できるかどうかが鍵:
       - A が joinMatch 済み → 受信可 → ゴースト消える ✓
       - A が joinMatch 前 → 受信不可 → ゴースト残る ★

※1 onPresenceJoin / onMatchPresenceJoin 内で sendInitPos を呼ぶ
    → B2 がまだ joinMatch 前でも OP_INIT_POS(A) は送信される
    → B2 は onmatchdata 登録前なので受信できない可能性あり
    
※2 pagehide の leaveMatch が成功した場合は即座（t=0.1で消える）
    失敗した場合は pong タイムアウト（ping_period 4s + pong_wait 6s = 最大10s）
```

## 根本原因

1. **iOS Safari の `pagehide`/`beforeunload` で `leaveMatch` が確実に送信されない**
   - WebSocket フレームが送出される前にページが破棄される

2. **クライアントが joinMatch 前にサーバーの MatchLeave を受信できない**
   - `onmatchdata` ハンドラは `joinMatch` 前に登録されるが、
     `onmatchpresence` は `joinMatch` 後に登録される
   - `joinMatch` 前に発生した MatchLeave は受信できない

3. **同一ユーザーIDでもセッションIDが異なる**
   - iPhone Safari のリロードで新しいセッションIDが生成される
   - サーバー側の `MatchJoin` 重複キックは MatchLeave と MatchJoin のタイミング次第で空振りする

## 対策の選択肢

### A. サーバー側で解決（推奨）

- **A1**: `MatchLoop` 内で定期的に接続状態をチェックし、切断済みセッションをキック
- **A2**: `getWorldMatch` RPC で返すプレイヤーリストに、マッチの現在プレゼンスのみを含める
  （切断済みだがまだ MatchLeave 未処理のセッションを除外）

### B. クライアント側で解決

- **B1**: `joinMatch` 後に `match.presences` と `remoteAvatars` を照合し、
  プレゼンスに含まれないアバターを削除（数秒遅延で実行）
- **B2**: `navigator.sendBeacon` + サーバーRPC でページ離脱時に確実に通知
