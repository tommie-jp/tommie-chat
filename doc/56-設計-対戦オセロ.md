# 対戦オセロ設計メモ

## コンセプト

部屋の中でオセロ対戦ができる機能。チャットSNSのコンセプトに合わせ、
既存の部屋内に「オセロ卓」を置く形で実装する。

## 方針：パネル＋ブロックのハイブリッド

- **操作・判定はパネル（HTML）** で行い、対局者はパネルで打つ
- **ブロックにも同期反映** して、部屋にいる人が 3D 空間でも盤面を見られる
- まずパネルだけ作り、動いてからブロック同期を追加する（段階的）

| 方式 | メリット | デメリット |
| --- | --- | --- |
| パネル主体 | タッチ／クリックで直感的、モバイル対応しやすい | 3D 空間との一体感は薄い |
| ブロック主体 | 没入感が高い、観戦が自然 | 操作しづらい、カメラ制御が必要 |
| **ハイブリッド** | 両方の利点を取れる | 実装量は増える |

## アーキテクチャ

```text
┌─ クライアント ──────────────────────────────────┐
│  OthelloPanel.ts (新規)                         │
│    - 8x8 HTML テーブル + タッチイベント          │
│    - 合法手ハイライト、スコア表示、パス/投了     │
│                                                  │
│  NakamaService.ts                                │
│    - RPC: othelloCreate, othelloMove, othelloJoin │
│    - Opcode: OP_OTHELLO_UPDATE (18?)             │
└──────────────┬───────────────────────────────────┘
               │ WebSocket
┌──────────────▼───────────────────────────────────┐
│  nakama/go_src/main.go (追記)                     │
│    - othelloCreate RPC  → ゲーム作成、Storage保存  │
│    - othelloMove RPC    → 合法手判定、盤面更新     │
│    - othelloJoin RPC    → 対戦参加                 │
│    - OP_OTHELLO_UPDATE  → 対局者＋観戦者へ配信     │
│    - 盤面は int8[64] で管理 (0=空, 1=黒, 2=白)    │
└──────────────────────────────────────────────────┘
```

## ゲームステート（Storage）

```text
キー: othello_<gameId>

{
  board: [64]int8,       // 盤面（8x8 を 1次元で管理）
  black: "user_id_1",    // 黒プレイヤー
  white: "user_id_2",    // 白プレイヤー
  turn: 1,               // 1=黒, 2=白
  status: "playing",     // waiting / playing / finished
  lastMove: {x:3, y:4},
  createdAt: "..."
}
```

## 対戦フロー

```text
プレイヤーA: othelloCreate RPC → gameId 取得、盤面パネル表示（待機中）
プレイヤーB: othelloJoin RPC  → 対戦開始、両者に OP_OTHELLO_UPDATE 配信
毎ターン:   othelloMove RPC(gameId, x, y) → サーバ判定 → 盤面更新 → 配信
終了:        パス2連続 or 盤面埋まり → status=finished → 結果表示
```

## 部屋との関係

- 既存の部屋の中に「オセロ卓」を置く（専用部屋ではなく）
- チャットしながら対戦できる → チャットSNSのコンセプトに合う
- 1 部屋に複数卓を置けるとゲームセンター的な雰囲気が出る

## サーバ側ロジック（Go）

### 合法手判定

- 8 方向（上下左右＋斜め 4 方向）に探索
- 相手の石を挟める場所だけが合法手
- 合法手がない場合は自動パス（サーバ側で処理）
- パス 2 連続で終局

### RPC 一覧（追加分）

| RPC 名 | 引数 | 戻り値 | 説明 |
| --- | --- | --- | --- |
| `othelloCreate` | `{worldId}` | `{gameId, board}` | 卓作成、待機状態 |
| `othelloJoin` | `{gameId}` | `{board, black, white}` | 対戦参加、ゲーム開始 |
| `othelloMove` | `{gameId, x, y}` | `{board, turn, flipped[]}` | 石を置く |
| `othelloResign` | `{gameId}` | `{winner}` | 投了 |

### Opcode（追加分）

| Opcode | 名前 | 方向 | 内容 |
| --- | --- | --- | --- |
| 18 | `OP_OTHELLO_UPDATE` | S→C | 盤面更新（対局者＋観戦者へ配信） |

## クライアント側（TypeScript）

### OthelloPanel.ts（新規）

- 8x8 の HTML テーブルで盤面描画
- タッチ／クリックで石を置く
- 合法手をハイライト表示
- スコア（黒○個 / 白○個）表示
- パス／投了ボタン
- 既存パネルパターンに合わせる（ドラッグ移動、最小化、Cookie 位置保存）

### 対戦招待 UI

- チャットコマンド `/othello @名前` で招待
- または部屋内のオセロ卓オブジェクトをクリック

## ブロック同期（フェーズ 2）

- `othelloMove` 処理時に盤面の差分をブロック更新としても送る
- 黒石＝黒ブロック、白石＝白ブロック、空＝緑ブロック
- 既存の `OP_BLOCK_UPDATE` をそのまま使える

## 実装フェーズ

| フェーズ | 内容 |
| --- | --- |
| **1** | Go 側：オセロロジック（合法手判定・裏返し・勝敗） |
| **2** | Go 側：RPC 3 本（create/join/move）＋ opcode 追加 |
| **3** | TS 側：OthelloPanel.ts — 8x8 盤面 UI ＋操作 |
| **4** | 対戦招待 UI（チャットコマンド `/othello @名前` 等） |
| **5** | ブロック同期 — 盤面をブロックで 3D 表現 |
| **6** | 演出（石を置く音、裏返しアニメーション、勝敗演出） |

## 懸念点・検討事項

- **メモリ**: 1 ゲーム 64 バイト程度、100 卓でも問題なし
- **切断対応**: 60 秒無操作でタイムアウト → 投了扱い
- **モバイル**: パネル方式なら既存のモバイル対応パターンがそのまま使える
- **パス判定**: 合法手がないときの自動パスはサーバ側で処理
- **ゲームロジックはサーバ必須**: クライアントで判定すると改ざんされる
