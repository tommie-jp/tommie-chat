# 検討事項: アメーバPigg 方向性メモ

tommChat が目指す方向性は **チャット主体のアバターSNS（アメーバPigg の立体版）**。
ゲーム性よりもコミュニケーションと自己表現が核。

---

## アメーバPigg とは

CyberAgent が2009年に開始した仮想空間アバターコミュニティ。
自分に似たアバター（ピグ）を作り、渋谷・原宿などの仮想広場でチャットする。
PC版は2019年に Flash 終了に伴いサービス終了。スマホ版（PIGG PARTY 等）が現行。

---

---

## tommChat の強み（ピグとの差別化）

- **3D ボクセル空間** — ピグは2Dだが、tommChat は3Dブロック建築ができる
- **ブロック建築 = 部屋デコレーション** — 家具配置の代わりにブロックで自由に空間を作れる
- **大人数同時接続** — 1000人以上の同時接続を目標としたサーバ設計

## チャットSNS方向の機能優先度

| 優先度 | 機能 | Nakama 対応 | 備考 |
| ---- | ---- | ---- | ---- |
| 1 | エモート | opCode 追加 | 非言語コミュニケーションの基本 |
| 2 | アバターカスタマイズ | Storage API | 服・髪型・色の保存と同期 |
| 3 | フレンド機能 | Friends API | 友達追加・オンライン状態・DM |
| 4 | 複数エリア/部屋 | 複数 Match | 自分の部屋に招待、公共広場の複数化 |
| 5 | オンライン状態・通知 | Status / Notification API | ログイン通知、フレンドのオンライン表示 |
| 6 | ギフト・トレード | Storage API + RPC | アイテムの受け渡し |
| 7 | ミニゲーム | 別 Match として実装 | 釣り・カードゲーム等 |
| 8 | ガチャ/ショップ | Wallet API | 仮想通貨・アイテム購入 |

---

## 「自分の部屋」の実装方針

ピグの人気機能「マイルーム」は、tommChat ではブロック建築で実現する。

- 部屋 = 専用 Match（プレイヤーごとに作成）
- ブロックデータは Nakama Storage API に永続化
- 他プレイヤーを部屋に招待（Match ID を共有）
- 公共エリア ↔ マイルーム間の移動 UI
